# Filtro de Estudio v1.0.0 — Plan de Implementación

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Llevar la extensión a 1.0.0 incorporando todas las mejoras (grupos A–E) manteniéndola ligera y funcional, aislada de `main` (0.1.8) vía worktree.

**Architecture:** MV3 sin cambios estructurales (MAIN `page-bridge.js` + ISOLATED `content.js` + `filter-core.js` + `background.js` + popup). Cambios de comportamiento en `content.js`/`popup.js`, más CI y docs.

**Tech Stack:** JS vanilla, Manifest V3, Node `node:test` + `jsdom` para tests. Sin dependencias nuevas en runtime, sin bundler.

## Global Constraints

- Sin dependencias nuevas en runtime. `jsdom` solo en dev.
- Sin bundler ni paso de build. Se mantiene el fallback inline de `FILTER_CORE` (0.1.6).
- Permisos MV3 sin cambios: `storage`, `activeTab`.
- Idioma de UI: español. Estilo oscuro existente (#1f1f1f / #8ab4f8).
- Versionado: rama `version/1.0.0` en worktree; manifiesto `1.0.0`; nombre "Filtro de Estudio para NotebookLM (1.0 beta)" durante desarrollo.
- `npm test` debe seguir en verde tras cada tarea (23 existentes + nuevos).
- Cada tarea termina con commit en `version/1.0.0`.

## File Structure

- `content.js` — observer, debounce, storage por notebook, empty state, scanSourcesPanel, contadores, colapso, módulo compartido de chips.
- `popup.js` — usa módulo compartido; elimina reimplementación.
- `popup.html` — ajustes menores (collapse/contador si se muestran en popup).
- `manifest.json` — versión 1.0.0, nombre beta.
- `filter-core.js` — sin cambios previstos.
- `page-bridge.js`, `background.js` — sin cambios.
- `styles.css` — **eliminado** (Grupo A).
- Nuevos: `.github/workflows/test.yml`, `CONTRIBUTING.md`.
- Tests: `content.dom.test.js` ampliado; `shared.test.js` si se extrae módulo.
